Sandbox developer console

Use this console to test the STC e-invoicing integration flow through the sandbox invoice endpoints. It helps prepare device enrollment, submit signed UBL invoices for clearance or reporting, inspect responses, and copy matching curl commands.

  • Device enrollment
  • Canonical invoice hash
  • Clearance sandbox
  • Reporting sandbox
  • Request inspector

Sandbox behavior

The backend still validates XML schema, invoice hash, XAdES-BES signature, certificate chain, enrolled device, and taxpayer TIN rules. Sandbox mode skips invoice persistence and ICV/PIH chain-state mutation.

Signed invoices must contain a certificate issued through this server's enrollment flow.
